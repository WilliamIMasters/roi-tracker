================================================================================
  LOOT CRATE ROI TRACKER — PATCH NOTES
================================================================================

--------------------------------------------------------------------------------
  v0.5
--------------------------------------------------------------------------------

Bug Fixes
  - Undo delete button in the toast was broken due to a quote-escaping issue in
    the generated onclick handler; now works correctly
  - Dark mode is now the default theme; light mode can be toggled via the
    sun/moon button and the preference is saved across sessions
  - Dark mode now correctly darkens the page header gradient
  - Dragging a crate card no longer interferes with selecting text in inputs;
    only the grip handle (⠿) on the left of the header initiates a drag

Improvements
  - Codebase split from a single index.html into three files: index.html
    (structure), style.css (all styling), and app.js (all logic); no change
    to functionality

Code Audit Fixes
  - Confirm-delete button now correctly turns red (was using CSS variables in
    inline styles, which browsers do not resolve)
  - ROI calculation no longer produces Infinity when a crate has zero cost
  - Break-even chance is now only shown when a coin conversion rate is set,
    preventing a misleading points-vs-£ unit comparison
  - Copy-to-clipboard now shows an error toast if the browser denies access
    instead of silently doing nothing
  - Deleting a crate now cleans up its sort state so it cannot accumulate
    indefinitely across many add/delete cycles
  - Patch notes fetch now times out after 10 seconds, matching JSONBin calls
  - Toast background, card shadows, conversion bar text, and coin icon colour
    are all now dark-mode aware; no more hardcoded light-theme hex values
  - updateCrate and updateItem now validate field names against an allowlist,
    preventing prototype pollution via crafted input
  - esc() now returns an empty string for null/undefined instead of the
    literal text "null" or "undefined"
  - showToast action button now verifies the handler function exists before
    rendering it, preventing silent failures
  - Status refresh interval now skips hidden tabs and is cleaned up on
    page unload to prevent accumulation across multiple open tabs
  - Import now validates the shape of every crate and item before accepting
    the file, preventing silently malformed data from corrupting the app
  - JSONBin requests now time out after 10 seconds instead of hanging
    indefinitely; the header moves to an error state rather than sticking on
    "Saving…"
  - Negative values are no longer accepted for crate cost or item value; item
    chances are clamped between 0 and 100
  - API errors now show plain-English messages ("Wrong access key", "Bin not
    found", "Rate limit hit") instead of bare HTTP status codes
  - Crates whose item chances exceed 100% now show a warning icon on their
    results card and ROI bar to indicate the numbers may be inflated

New Features
  - Duplicate crate — button in the crate header clones the crate and all its
    items, inserting it directly below the original
  - Duplicate item — copy button on each item row duplicates it immediately below
  - Undo last delete — deleting a crate or item shows a 5-second toast with an
    Undo button to restore it
  - Sort items by column — click the Chance, Value, or EV contrib. column header
    to sort ascending or descending; arrows show the active sort direction
  - Break-even chance — each results card shows the cumulative probability of
    getting back at least the cost of opening the crate
  - Dark mode — moon/sun toggle in the header switches themes; preference is
    saved across sessions
  - Drag to reorder crates — drag any crate card to rearrange the list
  - Copy result to clipboard — button on each results card copies name, ROI and
    EV as a formatted string for pasting into Discord
  - Confirm before deleting a crate — Remove button requires a second click
    within 3 seconds to prevent accidental deletions
  - Last-saved timestamp — header status shows how long ago the bin was saved,
    refreshing every 30 seconds

Improvements
  - Version number is now defined in a single APP_VERSION constant in the
    script; both the header and footer read from it automatically so there is
    only one place to update when bumping the version

--------------------------------------------------------------------------------
  v0.4
--------------------------------------------------------------------------------

New Features
  - Expand All / Collapse All buttons above the crate list for quick management
    when working with a large number of crates
  - Patch Notes button in the page footer opens a modal displaying the full
    version history; content is fetched live from PATCHNOTES.txt

Improvements
  - Shared bins panel now shows a clear "Shared bins" / "My bins" section header
    to distinguish pre-configured bins from personally added ones
  - Default shared bins (read-only) now show a "Read only" or "Write access"
    badge so it is clear at a glance what level of access you have

Bug Fixes
  - Write key override input now appears correctly below the bin row rather than
    being squeezed into the side of the card


--------------------------------------------------------------------------------
  v0.3
--------------------------------------------------------------------------------

New Features
  - Collapsible crate cards — click the chevron on any crate to collapse it down
    to a compact summary row showing item count, cost, expected value, and ROI
  - JSONBin.io integration for shared crate data across multiple users
      - Shared bins: pre-configure read-only bins that appear for all users
        automatically with no setup required on their end
      - Personal bins: add your own bins via the Bins panel using an access key
        and bin ID from jsonbin.io
      - Write key override: any user can supply their own write-access key for a
        bin (including shared/read-only ones) stored locally in their browser
      - Bin status indicator in the header shows connected / saving / error state
      - Manual Save button appears in the header when connected to a writable bin
      - Active bin data is loaded automatically on page open

Security
  - Switched authentication from X-Master-Key to X-Access-Key throughout,
    following JSONBin best practice for access-scoped keys


--------------------------------------------------------------------------------
  v0.2
--------------------------------------------------------------------------------

New Features
  - Coin conversion rate bar at the top of the page — enter a £-per-coin rate
    and each crate's cost will show its real-world GBP equivalent alongside the
    point value
  - ROI calculations now use the converted GBP cost when a conversion rate is
    set, giving a meaningful like-for-like comparison
  - Import / Export — download your full crate setup as a JSON file to share
    with others; importing offers the choice to replace existing crates or merge
    alongside them
  - Remove button added to each item row

Visual
  - Full visual redesign: purple gradient header, Inter typeface, Tabler Icons,
    defined colour palette with proper shadows and border radii
  - Item values are now labelled in GBP (£) throughout; crate costs remain in
    points and are marked with a coin icon to make the distinction clear
  - Expected value chip on each crate card highlighted in purple
  - Table headers, buttons, and warning banners all restyled for clarity
  - Version number displayed in the page header

Bug Fixes
  - Percentage validation warning/error now updates live as you type into the
    chance field rather than only refreshing when adding or removing items


--------------------------------------------------------------------------------
  v0.1.0  —  Initial Release
--------------------------------------------------------------------------------

  - Add and remove loot crates with a custom name and point cost to open
  - Add and remove items per crate, each with a drop chance (%) and GBP value
  - Live expected value (EV) calculation per crate
  - Percentage validation: warns when item chances sum to less than 100 %,
    and flags an error when they exceed 100 %
  - ROI comparison section with ranked metric cards and proportional bar chart
  - All data persisted automatically in browser localStorage
