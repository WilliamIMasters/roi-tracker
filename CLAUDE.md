# ROI Tracker — Project Instructions

## Patch Notes

Whenever you make a change to the project, update `roi-tracker/PATCHNOTES.txt`.

- Add notes under the **current version heading** (check the version shown in the `<h1>` of `roi-tracker/index.html`)
- If a heading for the current version does not exist yet, create one above the previous version, following the same format as the existing entries
- Group entries under the appropriate sub-heading: **New Features**, **Improvements**, **Bug Fixes**, **Visual**, or **Security**
- Keep each entry to one or two lines — describe what changed and why it matters to the user, not implementation detail
- Do this as part of the same task, not as a separate follow-up
